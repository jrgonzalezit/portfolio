## Por qué lo hice

Trabajando en soporte técnico N1/N2 (Le Utthe, retail multi-sucursal) repetía a mano, ticket tras ticket, las mismas tareas: relevar el hardware de un equipo, chequear si la red estaba fallando y armar un inventario de software para auditoría. Este toolkit automatiza esas tres tareas en PowerShell, pensado para acelerar la **resolución en primer contacto (FCR)** y estandarizar la documentación de incidencias.

## Qué incluye

| Script | Qué hace |
|---|---|
| `Get-SystemInfo.ps1` | Genera un reporte del equipo: SO, hardware, memoria, discos y red. Ideal para relevar una PC o documentar un ticket. |
| `Test-NetworkHealth.ps1` | Diagnóstico rápido de conectividad (placa de red, gateway, DNS e internet) con resultado OK/FALLA por prueba. |
| `Export-SoftwareInventory.ps1` | Exporta a CSV todo el software instalado (nombre, versión, fabricante, fecha). Útil para auditoría e inventario. |

## Uso

```powershell
# Relevar un equipo
.\scripts\Get-SystemInfo.ps1
.\scripts\Get-SystemInfo.ps1 -ExportPath "C:\Reportes\equipo01.txt"

# Diagnosticar la red
.\scripts\Test-NetworkHealth.ps1
.\scripts\Test-NetworkHealth.ps1 -TargetHost "www.google.com"

# Inventariar software instalado
.\scripts\Export-SoftwareInventory.ps1
.\scripts\Export-SoftwareInventory.ps1 -OutFile "C:\Reportes\software.csv"
```

## Decisiones técnicas

- **CIM en vez de WMI directo**: los cmdlets `Get-CimInstance` son más rápidos y no requieren privilegios elevados para la mayoría de las consultas, a diferencia de `Get-WmiObject`.
- **Salida dual (consola + archivo)**: cada script funciona interactivo por defecto, pero acepta un parámetro de export para integrarse a un flujo de documentación de tickets sin pasos manuales extra.
- **Sin dependencias externas**: corre con PowerShell 5.1 (el que trae Windows 10/11 de fábrica), para no depender de instalar nada en equipos de usuarios finales.
