> **Nota para vos, Junior:** este post es una plantilla de ejemplo (`draft: true`),
> no se publica en producción. Sirve para ver cómo se ve un post real:
> encabezados, código, y el callout que estás leyendo ahora mismo.
> Cuando tengas algo real para contar, editá este archivo (o creá uno nuevo)
> y sacá el `draft: true` del frontmatter.

## Por qué un blog

Todavía no hay contenido real acá — es intencional. Este espacio está listo para
cuando quieras documentar lo que vas aprendiendo en la Tecnicatura, algún
troubleshooting interesante del día a día en soporte, o el proceso de construir
tu próximo proyecto.

## Cómo se ve un bloque de código

```powershell
Get-CimInstance -ClassName Win32_OperatingSystem |
  Select-Object Caption, Version, OSArchitecture
```

## Cómo agregar tu primer post real

1. Copiá este archivo en `content/blog/` con un nombre nuevo (el nombre del archivo es el slug de la URL).
2. Editá el frontmatter: `title`, `description`, `date`, `tags`.
3. Sacá `draft: true` (o ponelo en `false`) cuando esté listo para publicarse.
4. Escribí el contenido en Markdown/MDX debajo del frontmatter.

Nada más — no hace falta tocar componentes ni rutas.
